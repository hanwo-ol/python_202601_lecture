# 웹 크롤링?

# HTML 구조
## 1. HTML의 정의와 트리 구조 (DOM)

HTML(HyperText Markup Language)은 웹 페이지의 내용을 서술하고 구조화하는 마크업 언어입니다. 크롤링 시 웹 페이지는 단순한 텍스트의 나열이 아닌, 요소(Element)들이 계층적으로 배치된 트리 구조(Tree Structure)로 해석됩니다. 이를 DOM(Document Object Model)이라 합니다.

*   **계층 구조 (Hierarchy):** HTML은 태그 안에 다른 태그가 포함되는 구조를 가집니다.
    *   **부모 요소 (Parent):** 특정 태그를 감싸고 있는 상위 태그.
    *   **자식 요소 (Child):** 특정 태그 내부에 포함된 하위 태그.
    *   **형제 요소 (Sibling):** 같은 부모를 공유하는 동일 선상의 태그.
*   **크롤링 적용:** 데이터를 찾을 때 '특정 태그의 바로 아래 있는 자식 요소' 혹은 '특정 태그 옆에 있는 형제 요소'를 찾는 방식으로 접근합니다.

## 2. HTML 요소의 기본 문법

HTML 요소는 시작 태그, 속성(Attribute), 내용(Content), 종료 태그로 구성됩니다. 크롤러는 이 구성 요소 중 특정 부분을 추출합니다.

```html
<태그명 속성명="속성값">내용</태그명>
<!-- 예시 -->
<a href="https://www.example.com">링크 텍스트</a>
```

*   **태그명 (Tag Name):** 요소의 성격을 규정합니다. (예: `a`, `div`, `span`)
*   **속성 (Attribute):** 태그의 추가 정보를 제공합니다. 이름과 값의 쌍(`name="value"`)으로 존재합니다. (예: `href`, `src`, `id`, `class`)
*   **내용 (Content):** 태그 사이에 위치한 텍스트나 다른 태그입니다. 크롤링의 주요 수집 대상입니다.

## 3. HTML 문서의 기본 구조

웹 페이지는 항상 다음과 같은 기본 골격을 가집니다.

```html
<!DOCTYPE html>
<html>
    <head>
        <meta charset="UTF-8">
        <title>페이지 제목</title>
    </head>
    <body>
        <div id="content">
            실제 데이터가 위치하는 곳
        </div>
    </body>
</html>
```

*   **`<html>`:** 문서의 최상위 요소(Root Element)입니다.
*   **`<head>`:** 문서의 메타데이터를 담습니다. 화면에 직접 표시되지 않으나 인코딩 방식(`charset`), 문서 제목(`title`) 등을 포함합니다.
*   **`<body>`:** 실제 웹 브라우저 화면에 표시되는 모든 콘텐츠(텍스트, 이미지, 링크 등)가 포함됩니다. **크롤링의 주 탐색 영역입니다.**

## 4. Body 내 주요 태그 (데이터 수집 대상)

크롤링 시 자주 마주치게 되는 태그들을 기능별로 분류하여 정리하였습니다.

### 4.1. 구역 설정 및 그룹화 태그
데이터의 위치를 특정하기 위해 가장 많이 탐색하게 되는 태그입니다.

*   **`<div>` (Division):** 특별한 의미 없이 구역을 나누는 블록 레벨 컨테이너입니다. 레이아웃을 잡는 데 사용되며, 하위 요소들을 그룹화합니다.
*   **`<span>`:** 특별한 의미 없이 텍스트의 일부분이나 인라인 요소를 묶는 컨테이너입니다. `<div>`와 달리 줄바꿈이 일어나지 않습니다.
*   **`<header>`, `<nav>`, `<section>`, `<article>`, `<footer>`:** `<div>`와 기능은 유사하지만, 해당 구역의 의미(상단, 내비게이션, 본문 등)를 명시적으로 나타내는 시맨틱(Semantic) 태그입니다.

### 4.2. 텍스트 정보 태그
실제 추출하고자 하는 텍스트 데이터가 담긴 태그입니다.

*   **`<h1>` ~ `<h6>` (Heading):** 제목을 나타냅니다. 숫자가 작을수록(h1) 상위 제목이며 중요도가 높습니다. 기사 제목, 상품명 등을 수집할 때 주로 접근합니다.
*   **`<p>` (Paragraph):** 문단(본문)을 나타냅니다. 긴 텍스트 데이터가 포함되어 있습니다.

### 4.3. 목록형 데이터 태그
게시판 목록, 메뉴, 상품 리스트 등 반복되는 데이터를 담습니다.

*   **`<ul>` (Unordered List):** 순서가 없는 목록을 감싸는 부모 태그입니다.
*   **`<ol>` (Ordered List):** 순서가 있는(1, 2, 3...) 목록을 감싸는 부모 태그입니다.
*   **`<li>` (List Item):** `<ul>` 또는 `<ol>` 내부의 각 항목을 나타내는 자식 태그입니다. 반복 수집(Loop)의 기준점이 됩니다.

### 4.4. 표(Table) 데이터 태그
행과 열로 구성된 정형 데이터를 담습니다. 재무제표, 시간표, 통계 자료 등에 사용됩니다.

*   **`<table>`:** 표 전체를 감쌉니다.
*   **`<thead>` / `<tbody>` / `<tfoot>`:** 표의 머리글, 본문, 바닥글을 구분합니다.
*   **`<tr>` (Table Row):** 표의 가로 행을 나타냅니다.
*   **`<th>` (Table Header):** 표의 제목 셀(주로 굵은 글씨)을 나타냅니다.
*   **`<td>` (Table Data):** 표의 실제 데이터 셀을 나타냅니다.

### 4.5. 링크 및 미디어 태그
다른 페이지로 이동하거나 미디어 파일을 다운로드할 때 사용합니다.

*   **`<a>` (Anchor):** 하이퍼링크를 생성합니다.
    *   **주요 속성 `href`:** 이동할 페이지의 URL이 담겨 있습니다. 크롤러가 다음 페이지로 이동하기 위해 수집합니다.
*   **`<img>` (Image):** 이미지를 표시합니다.
    *   **주요 속성 `src`:** 이미지 파일의 경로(URL)가 담겨 있습니다. 이미지 다운로드를 위해 수집합니다.

### 4.6. 입력 양식 태그
검색어 입력, 로그인 등 서버와 상호작용할 때 사용됩니다.

*   **`<form>`:** 입력 양식 전체를 감싸는 태그입니다.
*   **`<input>`:** 사용자가 데이터를 입력하는 필드입니다. (속성 `type`에 따라 텍스트 박스, 체크박스 등으로 변함)
*   **`<button>`:** 클릭 가능한 버튼입니다.

## 5. 식별자 속성 (Select를 위한 핵심 속성)

원하는 데이터를 정확히 핀포인트(Pinpoint)하여 가져오기 위해 사용하는 HTML의 핵심 속성입니다.

*   **`id` (아이디):**
    *   **정의:** 문서 전체에서 **유일한(Unique)** 요소를 식별하는 이름입니다.
    *   **특징:** 한 페이지 내에 중복된 `id` 값을 가진 태그는 존재해서는 안 됩니다(표준 규격).
    *   **활용:** 특정 게시글 본문이나 특정 검색창 등 단 하나의 요소를 찾을 때 가장 효율적입니다.
    *   **예시:** `<div id="main-article">...</div>`

*   **`class` (클래스):**
    *   **정의:** 여러 요소에 **공통적으로** 적용되는 그룹명입니다. 주로 스타일(CSS)을 일괄 적용하기 위해 사용됩니다.
    *   **특징:** 한 페이지 내에 같은 `class` 값을 가진 태그가 여러 개 존재할 수 있습니다. 또한, 하나의 태그가 공백으로 구분된 여러 개의 클래스를 가질 수 있습니다.
    *   **활용:** 게시판의 글 목록, 상품 리스트, 댓글 목록 등 반복되는 패턴의 데이터를 한 번에 수집할 때 사용합니다.
    *   **예시:** `<li class="item product-best">...</li>` (item 클래스와 product-best 클래스를 동시에 가짐)
