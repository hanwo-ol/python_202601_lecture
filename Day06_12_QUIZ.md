# NumPy 개념 점검 OX 퀴즈 (15문항)

**Q1.** 파이썬 리스트와 달리, NumPy 배열(`ndarray`)은 하나의 배열 안에 정수(int)와 문자열(str)을 혼합하여 저장할 수 없다. (혼합 시 모두 문자열로 변환된다.)
**Q2.** 이미 생성된 `ndarray`에 새로운 요소를 추가하면, 기존 메모리 공간이 늘어나는 것이 아니라 새로운 배열이 생성된다.
**Q3.** `ndarray.ndim` 속성은 배열에 저장된 전체 요소의 개수를 반환한다.
**Q4.** `np.arange(1, 5)`를 실행하면 `[1, 2, 3, 4, 5]`가 생성된다.
**Q5.** 정수형(int)으로 설정된 배열의 특정 인덱스에 실수(float) `3.14`를 대입하면, 자동으로 배열의 타입이 실수형으로 변경된다.
**Q6.** 배열을 슬라이싱(`arr[0:5]`)하여 생성한 변수는 원본 배열의 **뷰(View)**이므로, 값을 변경하면 원본 데이터도 함께 바뀐다.
**Q7.** 불리언 인덱싱(Boolean Indexing)을 사용하면 원본 배열과 동일한 형태(Shape)가 유지된다.
**Q8.** 팬시 인덱싱(Fancy Indexing, 정수 배열 인덱싱)은 뷰(View)가 아닌 **복사본(Copy)**을 생성한다.
**Q9.** 형태(Shape)가 `(3, 3)`인 배열과 `(3,)`인 배열은 덧셈 연산이 불가능하다.
**Q10.** 형태가 `(4, 5)`인 배열에 대해 `sum(axis=0)` 연산을 수행하면 결과 배열의 형태는 `(4,)`가 된다.
**Q11.** `reshape(-1, 5)`는 원본 데이터의 개수가 20개일 때, 자동으로 `(4, 5)` 형태의 배열을 생성한다.
**Q12.** NumPy의 유니버설 함수(ufunc)를 사용한 벡터화 연산은 파이썬의 `for` 반복문을 사용하는 것보다 처리 속도가 빠르다.
**Q13.** `arr1 * arr2` 연산은 행렬 곱(Matrix Multiplication)을 수행한다.
**Q14.** `ndarray.T` 속성을 사용하여 전치(Transpose)를 수행하면 데이터의 물리적 메모리 순서가 즉시 재정렬된다.
**Q15.** 3차원 배열 `(2, 3, 4)`의 전체 요소 개수(`size`)는 9개이다.
